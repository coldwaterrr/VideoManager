# 去除棋盘格背景并提取核心图标计划

## 问题描述
用户提供的 `videomanager_icon.png` 图片中存在灰白相间的棋盘格背景（通常表示透明背景的占位符）。这个棋盘格背景被错误地包含在了应用图标和系统托盘图标中，导致视觉效果不佳。

## 当前图片分析
- **外层背景**：灰白相间的棋盘格图案（应去除，设为透明）
- **中间背景**：蓝色渐变圆角矩形（可选保留或去除）
- **核心图标**：半透明白色文件夹 + 播放按钮 + 多层窗口图标

## 解决方案

### 方案A：使用Python + PIL处理（推荐）
1. **安装依赖**：`pip install Pillow`
2. **编写处理脚本**：
   - 检测并移除灰白棋盘格背景，替换为透明
   - 可选：裁剪掉蓝色圆角矩形背景，仅保留核心白色图标
   - 保存为带透明通道的 PNG
3. **替换文件**：更新 `videomanager_icon.png`
4. **重新构建**：验证图标效果

### 方案B：使用Node.js + Sharp处理
1. **安装依赖**：`npm install sharp`
2. **编写处理脚本**：利用 Sharp 库处理图像
3. **替换并构建**：同方案A

### 推荐方案
采用 **方案A（Python + PIL）**，理由：
- Python 图像处理生态成熟
- 可以精确识别棋盘格图案（通过像素颜色分析）
- 代码简单，易于调整
- 无需修改项目现有依赖结构

## 具体实施步骤

### 步骤1：安装Pillow
```bash
pip install Pillow
```

### 步骤2：编写图像处理脚本
脚本功能：
- 加载原图
- 遍历像素，识别棋盘格特征（灰白交替）
- 将棋盘格区域设为透明
- 可选：进一步提取核心图标边界
- 保存新图片

### 步骤3：替换并构建
- 覆盖 `videomanager_icon.png`
- 清理旧构建目录
- 执行 `npm run build`
- 验证图标显示效果

## 预期效果
- 应用图标和系统托盘图标不再显示棋盘格背景
- 核心图标清晰可见
- 背景透明或与窗口背景融合自然
