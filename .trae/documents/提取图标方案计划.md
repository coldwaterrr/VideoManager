# 提取图标方案计划

## 背景
用户有一个背景透明的图标图片 (`videomanager_icon.png`)，但目前该图片实际上是一个包含蓝色渐变背景和图标内容的方形图片。用户希望只保留中间的图标部分（蓝色文件夹 + 播放按钮 + 窗口），去掉周围的蓝色圆角矩形背景。

## 当前图片分析
当前图片结构：
- 外层：蓝色渐变圆角矩形背景
- 中间：半透明白色文件夹图标
- 文件夹上：播放三角形按钮
- 文件夹后：多层窗口/文档图标

## 实施方案

### 方案A：使用Python + PIL图像处理
1. 安装 Pillow 库
2. 编写 Python 脚本自动裁剪并提取中间图标
3. 将提取后的图标替换现有文件
4. 重新构建验证

**优点：**
- 自动化处理
- 可以精确控制裁剪区域
- 脚本可复用

**缺点：**
- 需要安装额外依赖
- 裁剪精度依赖于边缘检测

### 方案B：使用ImageMagick命令行工具
1. 检查系统是否已安装 ImageMagick
2. 使用命令行自动提取图标区域
3. 替换现有文件
4. 重新构建验证

**优点：**
- 命令行处理，无需编写代码
- 强大的图像处理能力

**缺点：**
- 系统可能未预装 ImageMagick
- 需要额外安装

### 方案C：手动提供裁剪后的版本
1. 我直接提供裁剪建议
2. 用户手动裁剪后替换文件
3. 重新构建验证

**优点：**
- 用户完全控制裁剪效果
- 无需安装工具

**缺点：**
- 需要用户手动操作
- 效率较低

### 推荐方案
推荐方案A（Python + PIL），因为：
- Python 环境通常已存在（项目依赖 Node.js，但 PIL 安装简单）
- 可以精确控制提取区域
- 自动化程度高
- 如果失败可以回退到方案C

## 实施步骤（方案A）
1. 使用 `pip install Pillow` 安装图像处理库
2. 编写脚本自动识别并裁剪图标区域：
   - 检测图片中的非透明/非空白区域
   - 计算图标边界框
   - 裁剪并保存新图片
3. 替换 `videomanager_icon.png`
4. 重新构建验证

## 注意事项
- 裁剪后保持图片格式为 PNG 以保留透明背景
- 确保图标尺寸适合应用图标（建议 512x512 或 1024x1024）
- 如果自动裁剪效果不理想，提供手动调整选项
