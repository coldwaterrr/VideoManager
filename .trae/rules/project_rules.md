# VideoManager 项目规则

## 重要操作准则

### 1. 推送与发布规则（2026-04-17）
- **推送代码到远程分支（git push）必须经过用户明确同意**
- **创建 GitHub Release 必须经过用户明确同意**
- AI 可以在本地完成 git add 和 git commit，但不得擅自执行 push 或创建 release
- 构建产物（build-release/、build-v2/、dist/ 等）不应被推送到远程仓库

### 2. 构建产物管理
- 构建产物不纳入 git 版本控制
- 使用临时输出目录（如 `build-v2/`）避免与正在运行的进程冲突
- 构建完成后验证产物完整性和大小

### 3. 代码修改原则
- 不改变现有功能的前提下进行样式或重构
- 遵循项目已有的代码规范和风格
- 修改完成后进行必要的类型检查和构建验证
